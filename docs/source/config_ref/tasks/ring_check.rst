``ring_check``
==============

Parameters controlling pierced-ring checks and how detected piercings are resolved. The action taken depends on the pierced ring's segtype (see segtypes)

Single-valued attributes:

  * ``delete``: for a pierced LIPID ring, which segment(s) to delete: 'piercee' (the ring's segment), 'piercer' (the threading bond's segment), 'both' (piercee plus the piercer when it too is a lipid -- robust for inter-threaded lipids, which strain both molecules), or 'none' (no deletion; the piercing is reported and the build stops). Does not apply to glycan or protein piercings (default: piercee)

  * ``cutoff``: distance cutoff between bond and ring COM's for checking, Angstroms (default: 10.0)

  * ``segtypes``: list of segment types whose rings are checked for piercing -- there is no default, so specify it explicitly (e.g. [lipid, glycan, protein]); an empty list checks nothing. The action on a detected piercing depends on the pierced ring's segtype: a pierced LIPID ring is resolved by deleting the offending segment (see delete); a pierced GLYCAN ring is fatal (a glycan cannot be deleted without breaking the glycan tree -- rebuild, e.g. with a different seed or more minimization); a pierced PROTEIN side-chain ring (His/Phe/Tyr/Trp/Pro) is resolved by rotating the side chain (chi2, then chi1) until it clears

  * ``max_ring_size``: only rings of at most this many atoms are considered (keeps real 5/6-membered rings while skipping the giant chordless cycles a disulfide makes through the protein backbone) (default: 7)
